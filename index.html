Playlister Specification

Users

  • Guest
    • Does not have account
  • Registered User
    • Has account
  • Logged-In User
    • Has account and is logged in
User Interface Mockup Diagrams (PDF)

Create Account (2.1)
  • Guest
  • Enter information and press "Create Account" button
  • Adds account to database but does not log in
  • No two accounts can have the same email address
  • All usernames must be unique
  • Feedback on any thrown errors
 Login to Account (2.2)
  • Registered User
  • Go to login page, enter information, click "Login"
  • User is taken to home screen with playlists
  • Feedback on any thrown errors
 Logout of Account (2.3)
  • Logged-In User
  • Can be done on any application screen
  • Click on account circle, click "Logout"
  • Logs out user and returns to splash screen
 Use Site as Guest (2.4)
  • Guest
  • Click "Continue as Guest" on splash screen
  • Takes user to All Lists screen
  • User can search for lists by name and open playlists
  • User can view comments and play playlists
  • User can go to Users screen and search for playlists by username
  • Cannot navigate to the home screen, button is visibly and functionally disabled
Create Playlist (2.5)
  • Logged-In User
  • "New Playlist" button creates new playlist and opens it
  • New playlist is given "Untitled ?" name
    • ? counts up starting at 0
  • No two playlists by the same user can have the same name
    • New playlist creation
    • Renaming playlist
 Rename Playlist (2.6)
  • Logged-In User
  • Double click on playlist card turns the name into a textfield, initially with the playlist name
  • User can type in any name and hit enter to change the name
  • Error pops up if another list already has that name
  • Text field should remain open
 Edit Playlist (2.7)
  • Logged-In User
  • User selects unpublished playlist which opens it
  • Click "Add Song" button to add a song
  • Double click on song to open the Edit Song dialog
    • Title, Artist, YouTube ID
  • User can drag and drop songs to change order
  • User can press "undo" and "redo" buttons
    • Transaction stack is cleared every time a playlist is opened
 Publish Playlist (2.8)
  • Logged-In User
  • User can press "Publish" button on unpublished playlist to permanently publish playlist
  • Cannot be undone
  • Cannot be edited
  • Can be deleted
  • Can be found and liked/disliked by others
  • Can be duplicated, duplicate is unpublished
Duplicate Playlist (2.9)
  • Logged-In User
  • User can duplicate from Home screen, All Lists, or Users
  • New playlist is added to user's playlists with same name and same contents
  • If user already has playlist with the same name, a number is added to the duplicated playlist name
 Delete Playlist (2.10)
  • Logged-In User
  • Click "Delete Playlist" button
  • Confirmation modal opens with "confirm" or "cancel" options
  • If confirm, playlist is removed from database and screen is refreshed
 View Owned Playlists (2.11)
  • Logged-In User
  • User can click on "SORT BY" button to to sort playlists by:
    • By Creation Date (Old-New)
    • By Last Edit Date (Old-New)
    • By Name (A-Z)
 Search by Playlist (2.12)
  • Guest or Registered User
  • User goes to "All Lists" screen
  • User enters text in search text field and hits enter
  • All published playlists that contain the text by any user are loaded
  • User can select and play a playlist
  • If no playlist contains the search text, the list of playlists is empty
    • Also empty if user hits enter and search text is blank
Search By User (2.13)
  • Guest or Registered User
  • User navigates to Users screen
  • User enters text in search text field and hits enter
  • All published playlists by any user that contains the text are loaded
  • User can select and play a playlist
  • If no user contains the search text, the list of playlists is empty
    • Also empty if user hits enter and search text is blank
 Play Playlist (2.14)
  • Guest or Registered User
  • Clicking on a playlist highlights it and loads it into the player
  • User can press "play" to play the first song
  • User can press "pause" to pause the song
  • User can press "next" or "prev" to navigate songs in playlist
  • Current song being played is highlighted in the playlist view
  • All published playlists should be playable by all users
  • Unpublished playlists can only be played by the owner
 Comment on Playlist (2.15)
  • Logged-In User
  • User can click on "comments" tab to show comments for that song
  • User can press "Like/Dislike"
  • User can add their own comment in the text field and hit enter
  • Comments are refreshed after adding a comment
